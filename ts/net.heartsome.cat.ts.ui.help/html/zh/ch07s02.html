2. 导出项目

导出项目是指把一个项目导出为 ZIP 压缩包，方便团队成员之间交换项目，以实现项目从前一个工作流程传递到后一个工作流程、或将一个大项目分发给多个团队成员协同完成，并避免频繁地进行创建项目、输入项目信息、设置记忆库/术语库等重复劳动。下面分别对这两种应用场景下的导出项目方式进行描述：

流程传递

项目从前一个工作流程传递到后一个流程时，通常需要完整地把整个项目导出，步骤如下：

  1. 在项目视图选择要导出的项目；

  2. 点击项目菜单 > 导出项目；

    图 142. 导出整个项目


  3. 若在第一步没有选择项目，还可以在这个导出项目对话框中选择要导出的项目。若只想导出特定格式的文件，可以点击过滤器类型按钮，然后选择或输入文件后缀名；

    图 143. 导出项目对话框


  4. 点击浏览按钮，选择要保存的路径和文件名；

  5. 点击确定按钮开始导出选中的项目。

项目分发

需要将项目分发给多个团队成员的时候，为方便操作，建议先把要分发给不同人员的文件分别存放在不同的文件夹中，然后按如下步骤操作：

  1. 点击项目菜单 > 导出项目；

    图 144. 导出项目的部分文件


  2. 在导出项目对话框中，只选择该项目中要导出给第一个团队成员的文件/文件夹，包括项目中共同的参考文件等；

    图 145. 导出项目：部分文件


  3. 点击浏览按钮，选择要保存的路径和文件名；

  4. 点击确定按钮开始导出选中的项目，导出的项目包格式为 .hszip；

  5. 重复第 1～4 步，分别导出要分发给每个团队成员的文件。
